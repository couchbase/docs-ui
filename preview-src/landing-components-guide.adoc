== Content-Aware Filtering {#filtering}

The filtering system allows users to find relevant content by multiple criteria:

.Filter Component
[source,asciidoc]
----
++++
<div class="landing-filter" id="landing-filter">
  <div class="landing-filter__header">
    <div class="landing-filter__title">Filter Content</div>
    <button class="landing-filter__toggle" aria-expanded="true" aria-controls="landing-filter-groups">
      <span class="landing-filter__toggle-text">Collapse</span>
      <span class="landing-filter__toggle-icon">▼</span>
    </button>
  </div>
  <div class="landing-filter__groups" id="landing-filter-groups">
    <div class="landing-filter__group">
      <div class="landing-filter__group-title">Product</div>
      <div class="landing-filter__options">
        <label class="landing-filter__option" data-filter-type="product" data-filter-value="server">
          <input type="checkbox" value="server" name="product" style="display: none;">
          Couchbase Server
        </label>
        <!-- Add more filter options -->
      </div>
    </div>
    <div class="landing-filter__group">
      <button class="landing-filter__clear">Clear All Filters</button>
    </div>
  </div>
</div>
++++
----

*Key Attributes:*

* `data-filter-type`: Category of filter (product, topic)
* `data-filter-value`: Specific value to filter by
* Auto-generates filter state and applies to matching cards

== Interactive Cards {#cards}

Cards are the primary content containers with hover animations and filtering support:

.Interactive Card
[source,asciidoc]
----
[.landing-card.landing-card--interactive]
====
++++
<div data-product="server" data-topic="database" 
     onclick="window.location.href='/target-page.html'">
  <div class="landing-card__icon">
    <i class="icon fas fa-database"></i>
  </div>
  <div class="landing-card__content">
    <h3 class="landing-card__title">Card Title</h3>
    <p class="landing-card__description">Brief description of the content or feature.</p>
    <div class="landing-card__tags">
      <span class="landing-card__tag landing-card__tag--product-server">Server</span>
    </div>
    <div class="landing-card__actions">
      <a href="/learn-more" class="landing-card__link">Learn More</a>
    </div>
  </div>
</div>
++++
====
----

*Card Types:*

* `landing-card--interactive`: Clickable with hover animations
* `landing-card--featured`: Highlighted with gradient background  
* Standard card: Static content container

*Data Attributes for Filtering:*

* `data-product`: server, mobile, capella, etc.
* `data-topic`: database, analytics, search, mobile, etc.
* `data-tags`: Space-separated list of tags

== Best Practices {#best-practices}

++++
<div class="landing-grid landing-grid--cols-1">
  <div class="landing-card">
    <div class="landing-card__content">
      <h3 class="landing-card__title">📝 Content Guidelines</h3>
      <ul>
        <li><strong>Hero Descriptions:</strong> Keep under 160 characters for impact</li>
        <li><strong>Card Titles:</strong> Use action-oriented language (3-7 words)</li>
        <li><strong>Card Descriptions:</strong> Explain benefits, not just features (1-2 sentences)</li>
        <li><strong>Filter Labels:</strong> Use clear, non-technical terms</li>
        <li><strong>Alt Text:</strong> Always provide meaningful image descriptions</li>
      </ul>
    </div>
  </div>
  
  <div class="landing-card">
    <div class="landing-card__content">
      <h3 class="landing-card__title">🎯 UX Guidelines</h3>
      <ul>
        <li><strong>Progressive Disclosure:</strong> Start with essentials, allow drilling down</li>
        <li><strong>Logical Grouping:</strong> Group related content together</li>
        <li><strong>Clear Hierarchy:</strong> Use consistent heading levels</li>
        <li><strong>Actionable Content:</strong> Every card should have a clear next step</li>
        <li><strong>Performance:</strong> Use lazy loading for images below the fold</li>
      </ul>
    </div>
  </div>
  
  <div class="landing-card">
    <div class="landing-card__content">
      <h3 class="landing-card__title">📱 Responsive Design</h3>
      <ul>
        <li><strong>Mobile First:</strong> Design for smallest screen, enhance up</li>
        <li><strong>Touch Targets:</strong> Ensure buttons/links are at least 44px</li>
        <li><strong>Readable Text:</strong> Maintain 16px minimum font size</li>
        <li><strong>Loading Performance:</strong> Optimize images and use webp format</li>
        <li><strong>Test Thoroughly:</strong> Check on multiple devices and screen sizes</li>
      </ul>
    </div>

== Migration Guide {#migration}

Converting existing hardcoded landing pages:

1. **Replace static HTML** with AsciiDoc content structure
2. **Add filter attributes** to existing cards (`data-product`, `data-difficulty`, etc.)  
3. **Update CSS classes** to use new component system
4. **Add hero section** for better visual hierarchy
5. **Include interactive JavaScript** (`13-landing-interactions.js` loads automatically)
